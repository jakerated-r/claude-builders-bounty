# Weekly Dev Summary — n8n + Claude API

Automated weekly narrative summary of a GitHub repo's activity (merged PRs, closed
issues, commits), generated by the Claude API and delivered to Discord or Slack.

Built for **claude-builders-bounty #5** ($200, Opire/Stripe).

## What it does

Every Friday at 5pm, the workflow pulls the last 7 days of activity for a configured
repo via the GitHub Search API, sends it to Claude (`claude-sonnet-4-20250514`) for a
≤250-word narrative, and posts the result to your Discord or Slack webhook. Language is
switchable EN/FR.

## Setup (5 steps)

1. **Import** `n8n-weekly-dev-summary.workflow.json` (n8n → *Workflows* → *Import from File*).
2. **Create a GitHub credential** — n8n → *Credentials* → *Header Auth* → name it
   `GitHub API (Header Auth)`, header **Name** `Authorization`, **Value**
   `Bearer ghp_yourtoken` (a fine-grained PAT with public-repo read is enough). Map it on
   the three `GitHub: …` nodes.
3. **Create an Anthropic credential** — *Header Auth* → name it
   `Anthropic API (x-api-key)`, header **Name** `x-api-key`, **Value** `sk-ant-…`. Map it
   on the `Claude (Messages API)` node.
4. **Edit the `Config` node** — set `githubOwner`, `githubRepo`, `language` (EN/FR),
   `destinationType` (`discord` or `slack`), and `webhookUrl` (your channel webhook).
5. **Test, then activate** — click *Execute Workflow* once to confirm a message lands in
   your channel, then toggle the workflow **Active** for the weekly cron.

## Configurable variables (all in the `Config` node)

| Variable | Example | Notes |
|---|---|---|
| `githubOwner` | `n8n-io` | repo owner / org |
| `githubRepo` | `n8n` | repo name |
| `language` | `EN` | `EN` or `FR` |
| `destinationType` | `discord` | `discord` or `slack` |
| `webhookUrl` | `https://discord.com/api/webhooks/…` | channel incoming webhook |

## Email variant (optional)

To deliver by email instead of a webhook, replace the `Deliver (Discord/Slack)` node with
n8n's **Send Email** (SMTP) node and map `={{ $json.summaryText }}` into the body. Everything
upstream is unchanged.

## Notes

- The GitHub **Search** API is used for all three fetches so each returns a single
  `{total_count, items}` object — no item fan-out, clean linear graph.
- Claude is instructed to never invent activity not present in the data (honest summaries
  on quiet weeks).
